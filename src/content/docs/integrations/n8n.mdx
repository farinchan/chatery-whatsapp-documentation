## Overview

**n8n-nodes-chatery-whatsapp** is an **n8n community node** that integrates with Chatery WhatsApp API. It allows you to automate WhatsApp messaging, manage sessions, retrieve chat history, and handle groups directly from your n8n workflows.

## Installation

### Via n8n Community Nodes (Recommended)

1. Go to **Settings > Community Nodes**
2. Select **Install**
3. Enter `n8n-nodes-chatery-whatsapp` in the **Enter npm package name** field
4. Agree to the risks of using community nodes
5. Click **Install**

### Via npm

```bash
npm install n8n-nodes-chatery-whatsapp
```

## Credentials Setup

To use this node, you need to configure the **Chatery API** credentials:

| Field | Description |
|-------|-------------|
| **URL** | Your Chatery WhatsApp API host URL (e.g., `http://localhost:3000`) |
| **API Key** | Your API key for authentication (optional, depending on your setup) |

### Setting Up Credentials

1. In n8n, go to **Credentials**
2. Click **Add Credential**
3. Search for **Chatery API**
4. Enter your Chatery server URL
5. Enter your API key (if configured)
6. Click **Save**

## Available Nodes

This package includes two nodes:

### 1. Chatery WhatsApp Trigger

A webhook trigger node that listens for WhatsApp events from the Chatery API.

**Supported Events:**

| Event | Description |
|-------|-------------|
| `all` | Listen to all events |
| `message` | New incoming message |
| `message_ack` | Message acknowledgment (sent, delivered, read) |
| `message_revoke` | Message deleted/revoked |
| `presence` | Contact presence update (online, typing, etc.) |
| `qr` | QR code generated for authentication |
| `ready` | Session is ready and connected |
| `disconnected` | Session disconnected |
| `group_update` | Group information updated |
| `group_participants` | Group participants changed |

**Filter Options:**

| Option | Description |
|--------|-------------|
| **Session Filter** | Filter events by specific session ID |
| **Chat ID Filter** | Filter events by specific chat ID |
| **Ignore Status** | Ignore status/story updates |
| **Only From Me** | Only receive messages sent by you |
| **Only From Others** | Only receive messages from others |

### 2. Chatery WhatsApp

The main node for performing WhatsApp operations via HTTP requests. This node supports multiple resources and operations.

## Usage Examples

### Example 1: Auto-reply to Messages

Create an automated response system:

1. Add **Chatery WhatsApp Trigger** node with `message` event
2. Add **Chatery WhatsApp** node with **Send Text** operation
3. Connect the trigger to the send node
4. Configure the send node:
   - **Session ID**: Use expression `{{ $json.sessionId }}`
   - **Chat ID**: Use expression `{{ $json.data.from }}`
   - **Message**: Your reply message

### Example 2: Send Notification to Group

Send notifications to a WhatsApp group from any trigger:

1. Use any trigger (e.g., Webhook, Schedule, Email, etc.)
2. Add **Chatery WhatsApp** node
3. Select **Chat** resource and **Send Text** operation
4. Configure:
   - **Session ID**: Your session name
   - **Chat ID**: Your group ID (e.g., `123456789012345678@g.us`)
   - **Message**: Your notification message

### Example 3: Session Management Workflow

Manage WhatsApp sessions programmatically:

1. Use **Chatery WhatsApp** node
2. Select **Session** resource
3. Operations flow:
   - **Connect**: Start a new session
   - **QR Image**: Get the QR code for scanning
   - **Status**: Check connection status
   - **Delete**: Remove session when done

### Example 4: Broadcast Message to Multiple Contacts

Send the same message to multiple contacts:

1. Use a trigger node with contact list data
2. Add **Split In Batches** node to process contacts
3. Add **Chatery WhatsApp** node with **Send Text** operation
4. Map each contact to the Chat ID field
5. Add **Wait** node to avoid rate limiting

### Example 5: Group Member Management

Automate group member management:

1. Use a trigger (Webhook, Form, etc.) with member data
2. Add **Chatery WhatsApp** node
3. Select **Group** resource
4. Use operations like:
   - **Add Participants**: Add new members
   - **Promote Participants**: Make someone admin
   - **Remove Participants**: Remove members

## Webhook Setup for Trigger Node

To use the **Chatery WhatsApp Trigger** node:

1. Create a workflow with the trigger node
2. Activate the workflow
3. Copy the webhook URL from the node
4. Add this webhook URL to your Chatery session:

```bash
curl -X POST http://localhost:3000/api/whatsapp/sessions/mysession/webhooks \
  -H "X-Api-Key: your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-n8n-instance.com/webhook/xxx",
    "events": ["all"]
  }'
```

Or add it during session creation in your Chatery API.

3. **Rate Limiting**: Add delay nodes when sending bulk messages

## Troubleshooting

### Connection Issues

- Ensure your Chatery server is running and accessible
- Verify the URL and API key in credentials
- Check if the session is connected

### Message Not Sending

- Verify the chat ID format is correct
- Ensure the session is in "connected" status
- Check if the phone number is registered on WhatsApp

### Trigger Not Firing

- Verify the webhook URL is correctly configured in Chatery
- Check if the workflow is activated
- Ensure the events match your filter settings
